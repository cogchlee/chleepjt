# Upbit Auto Trading Bot

업비트(Upbit) Open API를 활용하여 가상자산(암호화폐)을 **24시간 무중단** 자동 매매하는 파이썬 기반 프로그램입니다.

---

## 📁 프로젝트 구조

```
tradeProject/
├── main.py          # 메인 진입점: 매매 루프, 시그널 처리, 종료 리포트 출력
├── trade_utils.py   # 유틸 모듈: 티커 수집, 현재가 조회, 파라미터 최적화(학습)
├── .env             # API 키 및 실행 모드 설정 (git 제외)
└── README.md
```

## ✨ 핵심 기능

- **다중 기준 선정 로직**: 업비트 KRW 마켓에서 **거래대금 상위 10개** + **전일 대비 상승률 상위 10개**를 추출하여 시장 주도주와 급등주를 동시에 공략합니다. (최대 20개 종목)
- **파라미터 자동 최적화 (학습 모델)**:
  - 프로그램 시작 시 각 코인별 최근 14일 OHLCV 데이터를 바탕으로, K값(0.1~0.9) × MA 기간(3~10일)의 모든 조합을 백테스트하여 **누적 수익률이 가장 높은 파라미터** 를 코인마다 개별 산출합니다.
  - 이후 **24시간마다 자동으로 재학습**하여 최신 시장 상황을 반영합니다.
- **매수 전략 (변동성 돌파 + 단/장기 이평선 상승 필터)**:
  - 목표가 = 오늘 시가 + (전일 고가 - 전일 저가) × 최적 K
  - 현재가가 목표가를 돌파 돌파할 것 **AND** 오늘 시가가 단기 '최적 MA선'과 '장기 20일 이평선(MA20)' 모두 위에 있을 것 (확실한 상승 추세에서만 진입)
  - 보유 가능 KRW의 20%씩 분할 매수
- **매도 전략 (익절 / 손절 / 트레일링 스탑)**:
  - **기본 익절/손절**: +5% 이상 수익 시 익절, −2.5% 이하 손실 시 손절
  - **트레일링 스탑 (Trailing Stop)**: 수익률이 +1.5% 이상에 도달하면 발동. 이후 고점 대비 -1.5% 하락할 경우 즉각 매도하여 수익을 보존 (휩쏘 방지)
- **24/7 무중단 실행**: `Ctrl+C` 또는 SIGTERM 신호가 올 때만 종료됩니다. (별도 타이머 없음)
- **강제 종료 시 결과 리포트**: 종료 시 전체 거래 이력(매수/매도 종목, 가격, 수익률), 보유 자산 시가 환산, 총 P&L(%) 을 콘솔에 출력합니다.
- **시뮬레이션 / 실전 매매 분리**: `.env` 파일의 `SIMULATION_MODE` 값 하나로 전환 가능.

## 🚀 시작하기 가이드

### 1단계: Upbit API 키 발급

실제 매매를 위해 업비트 Open API 키가 필요합니다.

1. [업비트 마이페이지 → Open API 관리](https://upbit.com/mypage/open_api_management) 접속
2. **"Open API 발급 받기"** 진행 (카카오 인증 필요)
3. 권한 항목: `자산조회` ✅ `주문조회` ✅ `주문하기` ✅ 반드시 체크
4. **IP 화이트리스트** 등록 권장 (보안)

### 2단계: 패키지 설치

```bash
pip install pyupbit python-dotenv
```

### 3단계: `.env` 설정

`tradeProject/.env` 파일을 생성하고 아래 내용을 입력하세요.

```env
# Upbit API Keys (실제 매매 시 필요)
UPBIT_ACCESS_KEY="본인의_ACCESS_KEY"
UPBIT_SECRET_KEY="본인의_SECRET_KEY"

# Email Configuration (정기 상태 메일 발송용)
EMAIL_USER="your_email@gmail.com"
EMAIL_PASSWORD="your_app_password"
EMAIL_RECEIVER="receiver_email@example.com"

# 실행 모드
# True  → 시뮬레이션 모드 (가상 자금 10,000원으로 매매 로직 검증, 실제 주문 없음)
# False → 실전 모드 (실제 계좌 자금으로 자동 매수/매도 주문 실행)
SIMULATION_MODE="True"
```

### 4단계: 봇 실행

```bash
python main.py
```

- **시뮬레이션 모드** (`SIMULATION_MODE="True"`): 실제 API 주문 없이 10,000원 가상 잔고로 매매 로직 및 수익률을 로그로 검증합니다.
- **실전 모드** (`SIMULATION_MODE="False"`): 업비트 실제 계좌로 자동 주문이 나갑니다. 충분한 시뮬레이션 테스트 후 전환하세요.

### 5단계: 종료

`Ctrl+C`를 누르면 현재 거래 사이클 완료 후 아래 내용을 출력하고 종료됩니다:

## ⚙️ 설정 값 (main.py 상단)

| 변수                | 기본값  | 설명                                         |
| ------------------- | ------- | -------------------------------------------- |
| `SIM_START_KRW`     | 100,000 | 시뮬레이션 시작 자본금 (KRW)                 |
| `TICKER_LIMIT`      | 10      | 거래량 및 상승률 각각 10종목 추출 (최대 20)  |
| `TRAIN_DAYS`        | 14      | 파라미터 최적화에 사용할 과거 일수           |
| `RETRAIN_HOURS`     | 24      | 자동 재학습 주기 (시간)                      |
| `BUY_RATIO`         | 0.20    | 매수 시 잔고 대비 투입 비율 (20%)            |
| `MIN_BUY_KRW`       | 2000    | 최소 주문 금액 (KRW)                         |
| `TAKE_PROFIT_RATIO` | 0.05    | 고정 익절 기준 (+5%)                         |
| `STOP_LOSS_RATIO`   | -0.025  | 고정 손절 기준 (−2.5%)                       |
| `TRAILING_ACTIVATE` | 0.015   | 트레일링 스탑 발동 기준 (+1.5% 이상 수익 시) |
| `TRAILING_DROP`     | 0.015   | 고점 대비 하락 기준 (−1.5% 하락 시 매도)     |

## 📈 전략 요약

```
① 종목 선정  : 거래량 상위 10개 + 상승률 상위 10개 코인 자동 추출 (중복 제외 최대 20개)
② 초기 학습  : 추출된 코인 × (K: 0.1~0.9, MA: 3~10일) → 개별 최적 변동성/MA 조합 산출
③ 24h 재학습 : 14일 데이터 기반으로 종목 재선정 및 재최적화 (순환매 대응)
④ 매수 조건  : 현재가 ≥ 목표가(시가 + 변동폭 × K_best) AND 시가 > MA_best AND 시가 > MA20
⑤ 매도 조건  : 수익률 ≥ +5%(익절) OR ≤ −2.5%(손절) OR 부분수익 후 고점대비 -1.5% 하락(트레일링)
```
